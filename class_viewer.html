详细描述

类：观察者 

1.初始化：
设置观察者的位置、前向向量、上向向量，通过右手定则计算右向向量
2.重设：
重设观察者的位置、前向向量、上向向量，通过右手定则计算右向向量
3.工作过程：
函数m_Updatem_ViewerVectors()通过给出的m_World_Up、m_Front
   计算出一个标准化的m_Front、m_Up、m_Right向量
函数Get_m_View_Matrix()通过m_Position、m_Up、m_Front、m_Right四个向量
   计算出shader需要的view矩阵并返回
4.实现的主要功能：
（1）旋转视角
     原理为：
     shader中有矩阵projection * view * model
     此处让view = qmTransmView * m_Temp_View_Rot * m_View
     其中qmTransmView记录了对视角的平移
     而对视角的旋转分别记录在m_Temp_View_Rot 和 m_View两个矩阵
     每次旋转过程中，更新m_Temp_View_Rot的x方向和y方向旋转量
     然后将m_View = m_Temp_View_Rot * m_View并将m_Temp_View_Rot重置为单位阵
     来解决万向锁问题
     原因：
     鼠标事件调用RotateView的频率很高、每次的步进值很小
     因而projection * qmTransmView * m_Temp_View_Rot * m_View * model
     将m_View * model看成一个整体，每次选小旋转时操作m_Temp_View_Rot
     而旋转后更新m_View * model整体的旋转状态，下次小旋转就变成相对于该状态的旋转
     进而消除因多次旋转顺序记忆导致的万向锁问题
     弊端是导致了两次额外的4*4矩阵的乘法计算开销，但是该开销其实很小
     采用四元数可以完美解决万向锁，但是开发者懒，不想改
（2）平移视角
     TransXYView()通过改变m_Position实现沿着m_Up和m_Right方向移动视角
     TransZView()通过改变m_Position实现沿着m_Front方向移动视角
     注意：函数名称中含有的XYZ，实质是三维屏幕坐标系
